
# 003. Testes

## 1. Contexto

O IS4 possui muitos testes unitários.

Estes testes são exaustivos e cobrem muitos cenários de cada serviço em específico.

Isto gera muitos mocks específicos para a implementação de cada cenário.

No entanto, no RoyalIdentity muitos serviços serão remodelados por completo.

Isto significa que não será possível copiar os testes unitários do IS4 para o RoyalIdentity.

## 2. Decisão

Os testes unitários do RoyalIdentity serão reescritos.

Será dado mais atenção a testes de integração, que testam a aplicação como um todo.

Todos os testes usarão dados em memória, sem depender de um banco de dados.

Eles devem ser escritos em C# com xUnit. Não devem depender de serviços externos.

## 3. Consequências

Os testes unitários do IS4 não serão reaproveitados, mas podem ser usados como referência.
