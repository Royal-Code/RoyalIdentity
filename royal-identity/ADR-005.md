
# 005. Usuários

## 1. Contexto

No IS4 não há um gerenciamento de usuários próprio.

Há uma integração com o ASP.NET Identity, que é um framework para gerenciamento de usuários da Microsoft.

Este framework foi refatorado para o AspNetCore.

O meio de customização dele é por extensão de classes e métodos.

Como o RoyalIdentity poderá ter customizações por Realm, o método de extensão pode não ser suficiente.

## 2. Decisão

O RoyalIdentity terá um gerenciamento de usuários próprio.

O AspNetCore Identity poderá ser usado como base para o gerenciamento de usuários, já que é um projeto aberto.

Os dados dos usuários e regras serão configuráveis por Realm.

Será possível criar novas entidades de usuário e regras específicas para cada Realm.

## 3. Consequências

Será necessário uma implementação maior para atingir o mesmo nível de funcionalidades do AspNetCore Identity.
