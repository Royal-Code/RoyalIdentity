
# 004. Telas em Razor

## 1. Contexto

O IS4 não tem telas por padrão. 
É possível criar um projeto funcional a partir de um Template do DotNet.

As telas são desenvolvidas em cshtml e controllers.

Também não há telas administrativas ou de configuração.

Não há um meio de usar uma instância pronta do produto como no caso do Keycloak.

## 2. Decisão

Novas telas serão criadas em Razor.

As telas deverão contemplar todo fluxo de autenticação.

Também deverão existir telas administrativas, para cadastro de usuários e configurações.

Outras telas como administração do Realm e do servidor.

O RoyalIdentity deverá ser entregue como um produto completo, pronto para produção em uma imagem docker.

## 3. Consequências

O RoyalIdentity será mais fácil de ser utilizado.

A personalização (customização) continuará sendo possível.

Será necessário criar APIs administrativas.
