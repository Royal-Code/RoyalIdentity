# 009. Testes de isolamento multi-realm

## 1. Contexto

O RoyalIdentity adiciona suporte a múltiplos realms sobre uma base originalmente inspirada no IdentityServer4, que não possui esse modelo de isolamento.

Falhas de isolamento entre realms podem causar vazamento de clientes, sessões, tokens, códigos de autorização, consentimentos ou eventos entre locatários.

O ADR-003 define a preferência por testes de integração, mas o suporte multi-realm exige uma linha mínima de cobertura específica.

## 2. Decisão

O isolamento multi-realm deve ser protegido por testes de integração.

Esses testes devem usar pelo menos dois realms distintos e validar que dados criados em um realm não são visíveis nem utilizáveis a partir de outro realm.

A cobertura mínima deve incluir:
- Clientes;
- Sessões e cookies de autenticação;
- Access tokens e refresh tokens;
- Authorization codes;
- Consentimentos;
- Revogação;
- Eventos associados a realm;
- Operações de ciclo de vida via `IRealmManager`.

Os testes devem usar storage em memória e não depender de serviços externos.

## 3. Consequências

Mudanças em stores, handlers, validators, factories e eventos precisam preservar o isolamento entre realms.

A suíte de testes de integração será maior, mas passa a cobrir uma propriedade essencial de segurança do produto.

Os testes devem evitar dependência de estado compartilhado entre casos, usando identificadores únicos ou fixtures isoladas quando necessário.

Novas funcionalidades realm-scoped devem incluir testes que demonstrem o comportamento entre pelo menos dois realms.
