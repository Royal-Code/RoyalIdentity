# 008. Gerenciamento de realms via IRealmManager

## 1. Contexto

O suporte a realms exige mais do que persistir objetos `Realm`.

Criar, atualizar, habilitar ou desabilitar realms envolve regras de domínio, como proteger realms internos, validar unicidade de identificadores de rota e garantir que a estrutura de dados associada ao realm seja inicializada.

Se callers manipularem diretamente o storage, essas regras podem ficar duplicadas ou serem ignoradas.

## 2. Decisão

Será criado um serviço `IRealmManager` como fronteira de domínio para operações de ciclo de vida de realms.

As APIs administrativas e demais callers de alto nível devem usar `IRealmManager` para criar, atualizar, habilitar e desabilitar realms.

O `IRealmStore` permanece como abstração de persistência, mas não deve concentrar regras de domínio que pertencem ao ciclo de vida do realm.

Realms internos não podem ser removidos ou desabilitados por operações administrativas comuns.

Ao criar um realm, a infraestrutura de storage necessária para seus dados isolados deve ser inicializada junto com o registro do realm.

## 3. Consequências

As regras de ciclo de vida de realm ficam centralizadas em um serviço de domínio.

APIs administrativas podem evoluir sem expor diretamente detalhes do storage.

Implementações de storage precisam garantir que criar ou remover um realm trate também os dados associados ao realm.

Testes de integração devem validar que realms criados pelo manager ficam imediatamente funcionais para operações realm-scoped.
